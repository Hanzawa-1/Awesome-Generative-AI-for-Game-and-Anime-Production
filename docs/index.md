# Awesome Generative AI for Game & Anime Production { .brand-title }

A curated, **self-updating** catalog of generative-AI / ML **tasks** for **game and anime
production** — organized as an _Area → Task_ tree. Each task lists open-source models & papers
plus the proprietary tools studios actually use.

[Browse the Catalog :material-arrow-right:](catalog/index.md){ .md-button .md-button--primary }
[How it works :material-arrow-right:](about.md){ .md-button }

## What's inside

- :material-image: **Image & 2D Art** — concept art, line-art colorization, anime upscaling
- :material-cube-outline: **3D Generation** — text/image-to-3D, meshes, textures, scenes
- :material-account: **Characters & Avatars** — generation, auto-rigging, skinning
- :material-run: **Animation & Motion** — mocap, retargeting, frame interpolation / inbetweening
- :material-movie-open: **Video** — generation, restoration, rotoscoping
- :material-music: **Audio** — TTS, voice cloning, singing synthesis, music & SFX
- :material-script-text: **Text, Narrative & Design** — NPC dialogue, quests, localization
- :material-book-open-page-variant: **Manga & Comics** — panel generation, layout, colorization

## How it stays fresh

A weekly GitHub Actions job runs an LLM research agent (arXiv · Hugging Face · GitHub ·
DuckDuckGo) that discovers new entries, validates and de-duplicates them, then opens a Pull
Request for human review. Nothing lands on the site without passing schema + link checks and a
human merge. See [About](about.md) for the methodology.

!!! tip "Contributing"
    Spot something missing or wrong? Edit `data/entries.yml` (open source) or
    `data/services.yml` (proprietary) and open a PR. The agent's merge never overwrites
    human-edited fields.
